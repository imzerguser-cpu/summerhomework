# 1학기 방탈출 놀이 (QR 인증제) 설계 문서

- 작성일: 2026-07-12 (v2: 문제 구성 전면 개편)
- 대상: 전교생 40명 (1학년4, 2학년2, 3학년5, 4학년11, 5학년4, 6학년14)
- 목적: 1학기 교과(국어/수학/과학/영어) + 줄넘기 인증제 내용을 5개 방(강당/VR실/도서관/컴퓨터실/영어체험실)을 도는 방탈출 형식으로 복습. 팀당 태블릿 1대로 QR을 스캔해 문제를 풀고, 정답을 태블릿에 직접 입력(또는 음성)하는 방식.

## 2. 방 구성 원칙 (v2 변경)

**변경된 원칙**: 방마다 과목을 하나로 고정하지 않는다. 대신 방마다 8문제(학년별 인원 가중치: 1·2·3·5학년 각 1문제, 4·6학년 각 2문제)를 배치하되, **한 방 안에서 저학년(1~2학년)과 고학년(3~6학년)이 서로 다른 종류의 활동을 하도록** 짠다.

- 저학년(1~2학년)이 그 방에서 **국어/수학 교과 문제**를 푸는 경우 → 고학년(3~6학년)은 그 방에서 **줄넘기 미션**(종목 다양하게 섞고, 학년 수준에 맞는 통과 기준)을 수행한다.
- 저학년이 그 방에서 국어/수학을 풀지 않는 경우 → 저학년은 **줄넘기 미션**(오래뛰기·양발모아뛰기 중 하나, 저학년 수준 통과 기준)을 수행하고, 고학년은 그 방의 교과 문제(국어/수학/영어/과학 중 하나)를 푼다.
- 사회 과목은 제외하고 **과학으로 통일**한다(VR실 문제를 전부 과학으로 구성).

이 원칙에 따른 방별 배치:

| 방 | 1~2학년 | 3~6학년 |
|---|---|---|
| 도서관 (R1) | 국어 | 줄넘기 (다양한 종목) |
| 컴퓨터실 (R2) | 수학 | 줄넘기 (다양한 종목) |
| 영어체험실 (R3) | 줄넘기 (오래뛰기/양발모아뛰기) | 영어 |
| VR실 (R4) | 줄넘기 (오래뛰기/양발모아뛰기) | 과학 |
| 강당 (R5) | 줄넘기 (오래뛰기/양발모아뛰기) | 국어 또는 수학 (학년별 자기 수준) |

**알려진 트레이드오프**: 방 5개 × 학년 배치 구조상, 3학년과 5학년(각 1문제 슬롯)은 국어·수학 두 과목을 모두 배정받을 수 없다. 이번 구성에서는 **3학년은 수학만, 5학년은 국어만** 배정했다(둘이 겹치지 않게 분산). 두 과목을 모두 다루고 싶다면 보너스 문제(9번째 문제)를 추가하는 방법이 있다 — 필요하시면 알려주세요.

## 3. 문제 진행 방식 (QR / 정답 확인 메커니즘)

- 각 방은 8문제 **체인 구조**: 방 입구의 QR(R{n}Q1)을 스캔하면 1번 문제가 뜨고, 정답을 맞히면 "다음 QR이 숨겨진 위치" 힌트가 뜬다. 그 힌트를 보고 방 안을 돌아다니며 다음 QR(R{n}Q2)을 찾아 스캔 → 반복 → 8번까지 완료.
- QR은 문제 **ID만** 담는다 (예: `R3Q5`). 문제 텍스트/정답을 나중에 수정해도 QR을 다시 인쇄할 필요 없음.
- 순서 검증: 팀이 아직 풀지 않은 앞 번호가 있는데 뒷 번호 QR을 스캔하면 "아직 못 열어요, N번 QR부터 찾아보세요" 안내. 이미 푼 문제의 QR을 다시 스캔하면 정답을 복습용으로 재표시(진행에는 영향 없음).
- **학년 확인 절차 (신규)**: 문제 내용을 보여주기 전에 먼저 "⚠️ 이 문제는 OO학년 친구만 풀 수 있어요! 다른 학년 친구들은 힌트를 주지 말고 응원만 해주세요"라는 경고 화면을 띄우고, "이해했어요, 문제 보기" 버튼을 눌러야 실제 문제가 나타난다. (문제 데이터의 `grade` 필드를 이용해 자동 생성)
- 정답 판정: 앞뒤 공백/줄바꿈만 무시하고 나머지는 정확히 일치해야 정답 처리(맞춤법 학습 효과 유지). 영어 음성 문제는 음성 인식 결과 텍스트를 같은 방식으로 비교.
- **오답 시 재시도 대기시간 (신규)**: 같은 문제를 틀릴 때마다 다음 시도까지 기다려야 하는 시간이 늘어난다 — 1번째 오답 10초, 2번째 20초, 3번째 30초, 4번째 40초, 5번째 50초, 6번째부터는 60초 고정. 대기 중에는 제출 버튼이 비활성화되고 남은 시간이 카운트다운으로 표시된다. (같은 문제에서만 누적되며, 정답을 맞히면 초기화)
- 방 하나(8문제) 완료 시 그 방의 "황금열쇠 조각"(글자 1개)을 획득. 5개 방을 모두 완료하면 5글자가 모여 완성 문구가 뜨는 최종 화면 표시.
- **줄넘기 미션 판정 (v2 단순화)**: 보조 교사/도우미가 현장에서 통과 기준 충족 여부를 직접 확인해주고, 통과하면 팀은 화면에 뜬 확인 문구(예: "선생님께 확인받았으면 `성공`이라고 입력하세요")에 따라 고정 키워드를 입력한다. (구버전에서 쓰던 급수표 점수 조회·합산 방식은 QR/텍스트 입력 구조로 바뀌면서 더 이상 필요하지 않아 제거함)
- **영어 음성 문제**: math.html의 음성 인식 방식과 동일하게 마이크 버튼을 눌러 말하면 인식되는 구조. 한국어 낱말을 보여주고 그에 해당하는 영어 단어를 말하도록 한다.
- **5학년 영어 심화 (랜덤 출제)**: 미리 정한 직업 단어 목록(한글-영어 쌍) 중 하나를 매 팀마다 무작위로 뽑아 한글 직업명을 보여주고, 영어 철자를 입력하게 한다. 한 팀이 이 문제를 다시 열어도 최초 뽑힌 단어가 유지된다(재접속 시에도 동일 문제 유지).

## 5. 아키텍처

- 정적 파일 2개, GitHub Pages 배포: `escape.html`(팀용), `escapeAdmin.html`(교사용)
- 저장소: `imzerguser-cpu/escape` (사용자가 생성 완료, Pages 활성화 필요 시 안내)
- Firebase Realtime DB: 기존 jumprope 프로젝트(`jump-rope-43833`)에 새 경로 `escape_data` 추가, 실시간 진행상황 동기화 + localStorage 오프라인 백업(jumprope.html과 동일 패턴)
- QR 디코딩: `jsQR`(CDN) + `getUserMedia` 카메라 스트림
- QR 인코딩(관리자 인쇄 페이지용): 경량 QR 생성 라이브러리(CDN 또는 인라인)
- 음성 인식: Web Speech API(`SpeechRecognition`), math.html과 동일한 브라우저 지원 방식(미지원 브라우저는 키보드 입력 대체)
- 문제 40개는 `escape.html` 내 JS 배열로 하드코딩 (기존 SR/EV/TABLE_META 패턴과 동일). 각 문제는 `{id, grade, type, prompt, answer, hint}` 구조이며, `type`은 `text`(기본 텍스트 정답) / `voice`(음성 정답) / `random-pool`(팀별 무작위 출제, `pool` 배열 포함) 중 하나.

### 데이터 모델 (Firebase)
```
escape_data/
  progress/
    team1/
      R1: { solvedIdx: 3, solved:[true,true,true,false,false,false,false,false], wrongCounts:{R1Q4:2}, randomPicks:{} }
      R2: {...}
      R3: { ..., randomPicks:{ R3Q6: "요리사" } }
      R4: {...}
      R5: {...}
    team2..team5: 동일 구조
```
- `wrongCounts`: 문제별 누적 오답 횟수(재시도 대기시간 계산용)
- `randomPicks`: `random-pool` 유형 문제에서 그 팀에게 뽑힌 항목을 고정 저장(재접속해도 동일 문제 유지)

## 6. 팀용 앱 (escape.html) 화면 흐름

1. 시작화면: 조(1~5조) 선택 → localStorage에 저장, 이후 자동 진입
2. 홈: 5개 방 아이콘 + 완료 여부 + 모은 황금열쇠 조각
3. "QR 스캔" → 카메라 화면 → QR 인식 → ID 파싱 → 순서 검증
4. **학년 확인 화면** → "이 문제는 OO학년 친구만 풀 수 있어요" 경고 + "이해했어요, 문제 보기" 버튼
5. 문제 표시(텍스트 또는 음성 마이크 버튼) → 답 입력/발화 → 제출 → 판정
   - 정답: 축하 효과 + 다음 QR 힌트 노출 + 진행상황 저장(오답 카운트 초기화)
   - 오답: 정답 비노출, 오답 횟수에 따른 대기시간(10→20→30→40→50→60초) 카운트다운 후 재시도 가능
6. 방 8문제 완료 → 황금열쇠 조각 획득
7. 5개 방 완료 → 최종 조립 화면("미션 성공!")

## 7. 교사용 앱 (escapeAdmin.html)

- 대시보드 탭: 5조×5방 진행률(x/8) 실시간 그리드
- QR 인쇄 탭: 40개 QR을 방별로 인쇄용 레이아웃(A4)으로 렌더링, 브라우저 인쇄로 출력
- 진행상황 초기화 버튼(리허설 후 실전 전 사용)

## 8. 오류 처리

- 카메라 권한 거부/미지원: 문제 ID 수동 입력 대체 수단 제공(순서 검증 동일 적용)
- 음성 인식 미지원 브라우저: 키보드 입력으로 자동 전환(안내 문구 표시)
- Firebase 연결 끊김: localStorage 우선 저장 → 재연결 시 동기화
- 동일 QR 중복 스캔: 이미 푼 문제로 처리, 진행에 영향 없음

## 9. 배포 절차

1. 저장소 `imzerguser-cpu/escape` 생성 완료(사용자 확인됨)
2. 저장소 Settings → Pages에서 배포 브랜치 활성화 필요 여부 확인
3. 이후 `escape.html`, `escapeAdmin.html`을 해당 저장소에 push
4. 최종 주소: `https://imzerguser-cpu.github.io/escape/escape.html`(팀), `.../escapeAdmin.html`(교사)

## 10. 테스트 계획

- 인쇄된(또는 화면에 띄운) QR을 실제 태블릿 카메라로 스캔 테스트
- 여러 브라우저 탭으로 조를 다르게 선택해 동시 진행 시 Firebase 동기화 확인
- 오프라인(비행기모드) 상태에서 진행 후 재연결 시 데이터 유실 없이 동기화되는지 확인
- 오답 대기시간 단계별(10~60초) 동작 확인
- 영어 음성 인식 정확도 확인 (조용한 곳 vs 시끄러운 방)
- 오답/중복 스캔/순서 위반 시나리오 각각 확인

## 11. 알려진 갭 (교사 확인 필요)

- **3학년은 수학만, 5학년은 국어만** 배정됨 (방 5개 제한으로 두 과목을 동시에 배정 불가). 두 과목 모두 원하면 보너스 문제 추가 논의 필요.
- 줄넘기 미션의 실제 통과 기준(개수/초)은 예시일 뿐이며, 학년 구성과 학생 수준을 고려해 체육 교사가 최종 조정 권장
- QR을 숨길 실제 물리적 위치와 힌트 문구는 각 방의 실제 구조를 보고 현장에서 확정 필요
- 6학년 영어(R3Q7, R3Q8)와 5~6학년 과학 일부는 실제 교과서 진도와 다를 수 있어 확인 권장 (3~4학년 영어는 제공해주신 대본 어휘를 사용함)
