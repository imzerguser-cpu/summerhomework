# 3·4·5·6학년 방학 학습장 통합 설계

## 배경

현재 두 개의 독립된 사이트가 운영 중이다.

- `https://imzerguser-cpu.github.io/summerhomework/` — 3학년 전용 (repo `imzerguser-cpu/summerhomework`, 로컬 `Downloads/summerhomework/index.html`)
- `https://imzerguser-cpu.github.io/56homework/` — 5·6학년 전용 (repo `imzerguser-cpu/56homework`, 로컬 `Downloads/summerhomework/index56.html`, GitHub 내용과 바이트 단위로 동일 확인됨)

두 사이트를 하나로 합쳐 3·4·5·6학년 전체가 `summerhomework` 링크 하나로 접속하게 만든다. 4학년은 신규 추가 학년으로, 첨부된 `4학년 명단.pdf`의 학생 9명을 사용한다.

## 조사로 확인된 기존 구조

- 두 사이트 모두 완전히 self-contained 단일 `index.html`이다. 영어 퀴즈는 `<script type="text/html" id="quizDocTemplate">`에 전체 퀴즈 앱(옛 `english.html`/`english56.html` 내용)이 그대로 박혀 있고, "영어" 버튼을 누르면 그 텍스트로 Blob을 만들어 `<iframe>`에 로드한다. 줄넘기는 각 index 안에 완전히 인라인으로 구현되어 있다(`jumprope.html`/`jumprope56.html`은 더 이상 실제로 로드되지 않는 구버전 잔재).
- 3학년용 엔진(`english.html` 유래 템플릿, 이하 **엔진 A**)의 `GRADES` 객체 안에 **"3"과 "4" 두 학년 단원 데이터가 이미 모두** 존재한다(각 11개 단원, 단어·표현·이미지·음성 포함). 엔진 A는 5·6학년 엔진과 동일하게 `selectGrade(gid)` 함수를 내부에 갖고 있다.
- 5·6학년용 엔진(`english56.html` 유래 템플릿, 이하 **엔진 B**)은 이미 `loadEngQuizFrame(gradeOverride)` 패턴으로 특정 학년을 강제 주입하고 내부 학년선택 화면을 숨기는 기능을 갖추고 있다(`index56.html:1136-1157`). 3학년 쪽 `loadEngQuizFrame()`은 이 기능이 없어 현재 3학년 학생이 영어 퀴즈에 들어가면 내부적으로 3학년/4학년을 고를 수 있는 화면이 노출된다(사실상 미사용 버그성 동작).
- 줄넘기는 이미 학년 무관 공용 모듈이며, Firebase 프로젝트 `summerhomework-35c31`의 `jumprope_data` 경로에 학생 id 기준으로 저장된다. 3학년(`kim/na/min/baek/shin`)과 5·6학년(`s01`~`s19`) id가 이미 겹치지 않게 되어 있다.
- 학습 기록/비밀번호는 같은 Firebase 프로젝트에 `mado3_records`, `mado3_passwords`, `mado3_teacher_password`(3학년, 기본값 `260301`)와 `mado56_records`, `mado56_passwords`, `mado56_teacher_password`(5·6학년 공용, 기본값 `2026`)로 이미 실데이터가 쌓여 있다. 5·6학년은 로그인 버튼만 학년별로 나뉘어 있을 뿐 비밀번호는 지금 공유 중이다.
- 3학년만 "맛있는 구구단"(곱셈 연습) 활동이 추가로 있다.

## 설계

### 1. 기반 파일 및 스타일

`index56.html`을 뼈대로 삼는다(이미 다학년 탭/필터 패턴을 갖추고 있어 확장이 더 쉬움). 5·6학년의 시각 스타일(색상 변수, 폰트, 카드 UI 등 "5,6학년 양식")을 그대로 유지하고, 여기에 3학년 전용 기능(구구단 모듈, 엔진 A, 3학년 Firebase 연동)을 이식해 최종적으로 `Downloads/summerhomework/index.html`을 교체한다. 옛 3학년 `index.html`, `index56.html`, `english.html`, `english56.html`, `jumprope*.html` 등은 병합 후 참고용으로만 남기고(또는 정리), 실제 배포 파일은 새 통합 `index.html` 하나다.

### 3. 과제 구성 (학년별 분기)

메뉴 화면 카드 목록을 학년에 따라 분기한다.

- 3학년: 영어, 구구단, 줄넘기 (3개, 기존 그대로)
- 4·5·6학년: 영어, 줄넘기 (2개)

### 4. 영어 퀴즈 라우팅

두 엔진 템플릿을 모두 인라인으로 유지한다 — 데이터를 옮기거나 지우지 않는다.

- 3·4학년 → 엔진 A(`GRADES["3"]`/`GRADES["4"]` 포함) 로드, `selectGrade('3')` 또는 `selectGrade('4')`를 부트스트랩 주입하고 내부 학년선택 화면·버튼을 숨긴다(엔진 B가 이미 하는 방식을 엔진 A `loadEngQuizFrame`에도 동일 적용).
- 5·6학년 → 엔진 B, 기존 방식 그대로 `selectGrade('5')`/`selectGrade('6')` 부트스트랩.

결과적으로 3학년 학생에게 더 이상 4학년 옵션이 노출되지 않고, 4학년 학생은 자동으로 4학년 단원만 풀게 된다.

### 5. Firebase 데이터 (마이그레이션 없음)

기존 경로는 절대 이름을 바꾸지 않는다(과거 기록 보존).

- 기록: `mado3_records`(유지), `mado4_records`(신규), `mado56_records`(유지, 5·6학년 공용 그대로)
- 학생 비밀번호: `mado3_passwords`(유지), `mado4_passwords`(신규), `mado56_passwords`(유지, 5·6학년 공용 그대로 — 담임 대시보드는 어차피 `teacherGradeFilter`로 자기 학년 학생만 필터링해서 보여주므로 저장소 분리는 불필요)
- 줄넘기: `jumprope_data`(유지, 이미 전 학년 공용, id 겹치지 않음 확인됨)
- 담임 비밀번호: `mado3_teacher_password`(유지, 기본값 `260301`), `mado4_teacher_password`(신규, 기본값 `2026`), `mado5_teacher_password`(신규 — 기존 `mado56_teacher_password`에서 분리, 기본값 `2026`), `mado6_teacher_password`(신규 — 마찬가지로 분리, 기본값 `2026`). 분리 시점부터 5·6학년 담임은 각자 독립적으로 비밀번호를 관리하게 된다.

### 6. 담임 로그인 & 권한 범위

"3학년/4학년/5학년/6학년 선생님 로그인" 4개 버튼을 둔다. 로그인하면 `teacherGradeFilter`가 해당 학년으로 고정되고:

- 오늘 현황 / 전체 현황 / 스트릭 테이블: 자기 학년 학생만 표시(3학년만 구구단 열 추가 표시).
- 비밀번호 관리 테이블: 자기 학년 학생 비밀번호만 조회·수정 가능.
- 담임 비밀번호 변경 폼: 자기 학년의 `madoN_teacher_password`만 변경 가능.

다른 학년 데이터에는 UI 상 접근 경로 자체를 두지 않는다(기존 5·6학년 방식과 동일한 신뢰 모델 — Firebase 인증이 아닌 비밀번호 기반 접근 제어라는 기존 한계는 그대로 유지).

### 7. 반 친구들 보기 / 우리반 랭킹

기존에 "반 친구들 기록 보기"(로그인 전 공개 버튼)와 "우리반 랭킹"(메뉴 내부)이 이미 같은 화면(`screen-class`)을 공유하고 있다. 이 화면의 학년 탭을 3·4·5·6학년 4개로 확장한다. 구구단 통계 열은 3학년 탭에서만 표시한다.

### 8. 배포

`Downloads/summerhomework` 저장소의 `index.html`을 교체 후 `main` 브랜치에 커밋·push한다. `imzerguser-cpu/56homework` 저장소는 건드리지 않는다. 배포 즉시 `https://imzerguser-cpu.github.io/summerhomework/`에 반영된다.

### 9. 전체 관리자

담임 4명(3·4·5·6학년)을 총괄 관리할 수 있는 "전체 관리자" 계정을 별도로 둔다.

- **로그인 위치**: 학년 선택 화면에 눈에 띄지 않는 작은 "🔐 전체 관리자" 버튼을 5번째로 추가한다(4개 담임 로그인 버튼과 별도, 더 작게). 누르면 담임 로그인과 동일한 패턴의 비밀번호 입력 화면으로 이동한다.
- **비밀번호**: 신규 Firebase 경로 `mado_admin_password`(기본값 `9999`, 로그인 후 관리자가 직접 변경 가능)로 관리하며, 4개 학년 담임 비밀번호와는 완전히 별개의 값이다.
- **권한 범위(비밀번호 관리 전용, 그 외 대시보드는 제공하지 않음)**:
  - 전체 학생(3·4·5·6학년 전원)의 비밀번호를 학년별로 묶어서 조회·변경할 수 있다(기존 담임용 "비밀번호 관리" 표를 4개 학년 전체로 확장한 형태 — 즉 `mado3_passwords`, `mado4_passwords`, `mado56_passwords`를 모두 구독해서 한 화면에 표시).
  - 4개 학년 담임 비밀번호(`mado3_teacher_password`, `mado4_teacher_password`, `mado5_teacher_password`, `mado6_teacher_password`)를 모두 조회·변경할 수 있다.
  - 오늘 현황 / 전체 현황 / 스트릭 같은 학습 기록 대시보드는 관리자 화면에 포함하지 않는다(요청 범위를 비밀번호 관리로 한정).
- 개별 학년 담임은 여전히 자기 학년 것만 보고 바꿀 수 있으며, 전체 관리자만 4개 학년을 한 화면에서 다룰 수 있다.

### 10. 검증 계획

자동 테스트가 없는 순수 정적 HTML(+Firebase) 프로젝트이므로 로컬 서버로 띄운 뒤 브라우저로 수동 스모크 테스트한다:

- 학년 4개 모두 선택 → 학생 목록 정상 표시
- 각 학년 학생 로그인(비밀번호 확인 포함)
- 4학년 영어 퀴즈 진입 시 4학년 단원만 나오는지, 3학년 진입 시 3학년 단원만 나오는지(내부 학년선택 화면이 더 이상 노출되지 않는지)
- 3학년만 구구단 카드가 보이는지, 4·5·6학년은 안 보이는지
- 줄넘기 기록 입력이 4학년에서도 정상 저장되는지
- 담임 로그인 4개 버튼이 각자 다른 비밀번호로 분리되어 있는지, 자기 학년 학생/담임 비밀번호만 변경되는지
- 반 친구들 보기 / 우리반 랭킹에서 4개 탭 전환이 정상 동작하는지
- 전체 관리자 로그인 후 4개 학년 학생 비밀번호와 4개 담임 비밀번호를 모두 조회·변경할 수 있는지, 학습 기록 대시보드는 노출되지 않는지

### 용량 참고

두 영어 엔진(이미지·음성 포함)을 모두 인라인으로 유지하므로 최종 파일은 약 11~12MB가 된다(현재 두 사이트 용량의 합과 비슷한 수준). 정적 페이지라 동작에는 문제없으나 최초 로딩이 다소 걸릴 수 있다.

## 사용자 결정 사항

- 4학년 영어 콘텐츠: 이미 엔진 A에 들어있는 데이터를 그대로 4학년용으로 사용(신규 제작 불필요).
- 담임 비밀번호 초기값: 3학년은 기존 `260301` 유지, 4·5·6학년은 모두 `2026`으로 통일.
- 전체 관리자: 비밀번호 관리 기능만 제공(학습 기록 대시보드 제외), 로그인 버튼은 학년 선택 화면에 작게 5번째로 추가.
